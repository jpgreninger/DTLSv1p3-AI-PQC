Detailed Description

Protocol Version Manager for DTLS v1.3.

Comprehensive version negotiation system handling DTLS protocol version negotiation, validation, and backward compatibility checks following RFC 9147 specifications.

Key Features:
  • üîÑ Bidirectional Version Negotiation: Client and server-side version handling
  • üõ°Ô∏è Security-First Design: Built-in downgrade attack detection
  • üîß Backward Compatible: Seamless DTLS v1.2 fallback support
  • ‚ö° High Performance: Optimized for production environments
Security Note:
This class implements critical security measures including version downgrade attack detection as specified in RFC 9147 Section 4.1.3.
Example:\n
// Server-side version negotiation
VersionManager vm;
auto result = vm.negotiate_version_from_client_hello(client_hello);
if (result.is_ok()) {
auto negotiated = result.value().negotiated_version;
// Apply to ServerHello...
}

◆ negotiate_version_from_client_hello()

Result< NegotiationResult > dtls::v13::protocol::VersionManager::negotiate_version_from_client_hello ( const ClientHello &  client_hello) const

Server-side version negotiation from ClientHello.

Analyzes the ClientHello message to determine the optimal protocol version for the connection, considering supported versions extension and legacy compatibility.

Parameters
client_helloThe received ClientHello message containing version preferences
Returns
NegotiationResult containing negotiated version and security analysis
Complexity:
O(n*m) where n=client versions, m=server versions
Performance Note:
Optimized for typical negotiation scenarios with early exit strategies
Security Note:
Implements mandatory downgrade attack detection per RFC 9147 Section 4.1.3
Example:\n
VersionManager server_vm;
auto result = server_vm.negotiate_version_from_client_hello(client_hello);
if (result.is_ok() && !result.value().version_downgrade_detected) {
// Safe to proceed with negotiated version
auto version = result.value().negotiated_version;
}
